Detailed Description

Structures which represent a rig-definition file (1:1 match)

Author
Petr Ohlidal
Date
12/2013 Values are initialised to their defaults, or 0 if there's no default.

NOTE ON ARCHITECTURE: These structs were designed for a single purpose - to bring data from rig def. file. Do not use them for any other purpose, don't modify them beyond their original purpose. Avoid modifying their contents, only Parser should do that; use them only for reading;

NOTES: Since these are open structs, the m_ prefix for member variables is not used. Members prefixed by _ are helper flags which mark special values or missing values.
